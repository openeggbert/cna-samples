# NEXT.md — cna-samples handoff document

## 1. Project summary

**cna-samples** is a CMake C++ project that ports the official Microsoft XNA Game
Studio 4.0 sample collection from C# to C++, running on
[CNA](../cna) — a C++ reimplementation of the XNA 4.0 programming model built on
SDL3 + EasyGL (OpenGL ES 3.x).

**Main goal:** provide a complete, runnable C++ equivalent of every applicable XNA
4.0 sample, serving as both living integration tests for the CNA framework and a
reference for developers migrating XNA/MonoGame code to C++.

**Current phase:** Phase 1 (Foundation) — two samples exist and previously compiled
and ran, but a CNA upstream API change has broken the build.

**Key architectural decisions:**
- Each sample is an `add_subdirectory` under `samples/` with its own
  `CMakeLists.txt` using the `cna_add_sample()` helper from `cmake/SampleHelpers.cmake`.
- CNA is consumed as a sibling `add_subdirectory(../cna CNA_BUILD)` — no install step.
- All XNA C# public properties become `getXxxProperty()` / `setXxxProperty()` in CNA
  (generated by the `DEF_PROP` macro from sharp-runtime).
- XNB assets are not supported; raw formats (PNG, OGG, glTF) must be used instead.

## 6. Architecture notes

```
cna-samples/
├── CMakeLists.txt          ← root; add_subdirectory(../cna CNA_BUILD)
├── cmake/SampleHelpers.cmake  ← cna_add_sample(name SOURCES ...) macro
└── samples/
    └── SampleName/
        ├── CMakeLists.txt  ← calls cna_add_sample()
        └── src/
            ├── Program.cpp      ← main(); includes CNA/Entrypoint.hpp
            ├── SampleGame.hpp   ← derives from Microsoft::Xna::Framework::Game
            └── *.hpp            ← helper classes (inline, header-only per sample)
```

**CNA property access rule (critical):**
Every XNA public property `Foo.Bar` becomes `foo.getBarProperty()` / `setBarProperty()`.
Methods that *look* like properties use `()`: `CurrentTechnique()` became
`getCurrentTechniqueProperty()`. Always check the current header before using.

**Every `Game` subclass must override:**
```cpp
const std::string& GetTypeName() const override {
    static const std::string name = "ClassName";
    return name;
}
```

**Render state (NOXNA workarounds, no XNA-style setters yet):**
```cpp
device.SetDepthTestEnabled(bool);
device.SetBlendEnabled(bool);
device.SetDepthWriteEnabled(bool);
```

**Linking pattern (Linux/GCC/Clang with EasyGL):**
```cmake
target_link_libraries(target PRIVATE
    -Wl,--start-group CNA ${BACKEND_TARGET} -Wl,--end-group SHARP_RUNTIME)
```
Handled automatically by `cna_add_sample()`.

**Invariants:**
- `cna_add_sample()` in `cmake/SampleHelpers.cmake` is the single point for linking rules.
- Never add XNB loading — use raw asset formats only.
- CNA is always consumed as `add_subdirectory(../cna CNA_BUILD)` — never as an installed package.

## 7. Useful commands

```bash
# Configure (from repo root)
cmake -B cmake-build-debug -DCMAKE_BUILD_TYPE=Debug -DCNA_GRAPHICS_BACKEND=EASYGL

# Build everything
cmake --build cmake-build-debug -j4

# Build one sample
cmake --build cmake-build-debug --target cna_sample_primitives -j4
cmake --build cmake-build-debug --target cna_sample_primitives3d -j4

# Run PrimitivesSample (pre-built binary)
./cmake-build-debug/samples/PrimitivesSample/cna_sample_primitives

# Run Primitives3D (pre-built binary)
./cmake-build-debug/samples/Primitives3D/cna_sample_primitives3d

# Reproduce the current build error
cmake --build cmake-build-debug --target cna_sample_primitives 2>&1 | grep "error:"

# Check CNA Effect API

  ../cna/include/Microsoft/Xna/Framework/Graphics/Effect.hpp \
  ../cna/include/Microsoft/Xna/Framework/Graphics/EffectTechnique.hpp
```

## 9. Do not do yet

- **Do not port Phase 2–5 samples** until Phase 1 is clean and building.
- **Do not add XNB loading** — it will not be supported; use raw formats.
- **Do not refactor CNA** from cna-samples — fix only what breaks here.
- **Do not add a model loader** until CNA's glTF pipeline is designed.
- **Do not implement SpriteFont** inside cna-samples — it belongs in CNA.
- **Do not change the `cna_add_sample()` macro** without checking all existing sample
  `CMakeLists.txt` files.
- **Do not assume CNA API is stable** — always grep the current headers before using
  any method.
- **Do not create a new cmake-build-* directory** without first deleting the stale one.
