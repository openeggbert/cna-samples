# NEXT.md — CNA Samples Handoff

## 1. Project Summary

**What:** C++ ports of the official Microsoft XNA Game Studio 4.0 sample collection,
running on CNA — a C++ reimplementation of the XNA 4.0 API built on SDL3 + EasyGL
(OpenGL ES 3.x backend).

**Main goal:** Port all ~83 applicable XNA 4.0 desktop samples to CNA C++, preserving
the original class hierarchy and naming (`Microsoft::Xna::Framework::*`), serving as
living integration tests for the CNA framework and a migration reference.

**Current phase:** Phase 1 (Foundation — 12 samples) is complete. Phase 2 (2D Games)
has started — Platformer (#013) is the first Phase 2 sample done.

**Key architectural decisions:**
- One executable per sample; no shared sample library.
- All sample code is header-only (except `Program.cpp`) to avoid the need for
  separate translation units.
- Circular class dependencies resolved by forward-declaring and defining method
  bodies after all referenced classes are fully defined (see Platformer pattern).
- Assets: `Load<Texture2D>()` → PNG; `Load<SpriteFont>()` → `.font.json` + PNG atlas
  generated by `tools/make_font.py`; `Load<SoundEffect>()` → WAV only;
  `Load<Song>()` → MP3/OGG/WAV/WMA/FLAC.
- `CONTENT_DIR` argument to `cna_add_sample()` is mandatory for any sample that
  loads assets at runtime.

| Label | Description |
|-------|-------------|
| confirmed bug | **Primitives3D: DiffuseColor ignored.** `BasicEffect.setDiffuseColorProperty()` has no effect for `VertexPositionColor` geometry. ## 6. Architecture Notes

### Sample directory layout
```
samples/SampleName/
  src/
    Program.cpp          # int main() { Game g; g.Run(); return 0; }
    GameClass.hpp        # all logic inline (no .cpp unless circular deps)
    Helper.hpp           # additional classes, same header-only style
  Content/               # copied next to binary at build time via CONTENT_DIR
  CMakeLists.txt         # cna_add_sample(name SOURCES src/Program.cpp CONTENT_DIR ...)
  missing.md             # required: differences vs XNA original
  SampleName.htm         # required: original MS documentation, verbatim copy
```

### CNA API rules — must not break
- All XNA properties: `getXxxProperty()` / `setXxxProperty()` — never raw member access.
- Every `Game` subclass must implement:
  ```cpp
  const std::string& GetTypeName() const override {
      static const std::string name = "ClassName";
      return name;
  }
  ```
- `Content.Load<T>()` returns **T by value** — never take its address directly.
  Store in a value or `std::optional<T>`, then pass as `const T&`.
- `SpriteBatch::Draw` takes `const Texture2D&` — use `*optionalTex` or direct value.
- Virtual override signatures: `void Update(GameTime& gameTime)` and
  `void Draw(const GameTime& gameTime)` — note the const on Draw.
- No `operator+=` for `Vector2` or `TimeSpan` — write `v = v + expr`.
- `Color` has no default constructor — initialize at declaration or use ternary.
- `System::Random random_{seed}` for seeded random (sharp-runtime type, not std::mt19937).

### Circular dependency pattern (Platformer)
When classes reference each other (e.g. Level ↔ Player/Gem/Enemy):
1. Forward-declare all classes at top of each header.
2. Write class bodies using only pointers/optional members to forward-declared types.
3. Collect all method bodies that need the full type into the end of `Level.hpp`,
   after `Level` is fully defined. Mark them `inline`.

### SpriteFont generation
```bash
# Font directly in Content/
python3 tools/make_font.py /path/to/Font.ttf <size_px> samples/X/Content/FontName

# Font in a subdirectory (Content/Fonts/)
python3 tools/make_font.py /path/to/Font.ttf <size_px> \
    samples/X/Content/Fonts/FontName \
    --content-root samples/X/Content
```
`--content-root` is required for subdirectory fonts so the texture path in the JSON
is `"Fonts/FontName.png"` rather than just `"FontName.png"`.

### Sound assets
- `SoundEffect` loads `.wav` only. Convert with: `ffmpeg -i src.wma dst.wav`
- `Song` (music) loads: `.mp3 .ogg .wav .flac .opus .aac .wma`

## 7. Useful Commands

```bash
# Configure (from repo root, first time)
cmake -B cmake-build-debug -DCMAKE_BUILD_TYPE=Debug

# Build all enabled samples
cmake --build cmake-build-debug

# Build one specific sample
cmake --build cmake-build-debug --target cna_sample_platformer

# Run a sample
./cmake-build-debug/samples/Platformer/cna_sample_platformer
# Platformer controls: ←/→ or A/D = move, Space/↑/W = jump,
#                      Space or gamepad A = continue after death/win/loss

# Generate SpriteFont (font directly in Content/)
python3 tools/make_font.py \
    /usr/share/fonts/truetype/dejavu/DejaVuSans.ttf 19 \
    samples/MySample/Content/HudFont

# Generate SpriteFont (font in Content/Fonts/ subdirectory)
python3 tools/make_font.py \
    /usr/share/fonts/truetype/dejavu/DejaVuSans.ttf 19 \
    samples/MySample/Content/Fonts/HudFont \
    --content-root samples/MySample/Content

# Convert WMA sound effect to WAV
ffmpeg -i samples/MySample/Content/Sounds/Effect.wma \
          samples/MySample/Content/Sounds/Effect.wav

# Run all Phase 1 samples to confirm nothing is broken
for s in primitives primitives3d texturesandcolors spriteeffects spritesheet \
         shaperendering inputreporter inputsequence safearea generatedgeometry platformer; do
    timeout 3 ./cmake-build-debug/samples/*/cna_sample_${s} 2>&1 | grep -E "error|crash|terminate" || echo "$s: OK"
done
```

## 9. Do Not Do Yet

- **Do not start Phase 3 or 4** (3D shaders, models, animation) — `Content.Load<Model>()`
  is not implemented in CNA; would require XL effort in `../cna` first.
- **Do not port Spacewar** — it uses `Content.Load<Model>()`.
- **Do not port CameraShake** — also uses Model.
- **Do not port samples requiring SkinnedEffect or custom Content Pipeline types**
  (ReachGraphicsDemo, SkinningSample, etc.).
- **Do not refactor the header-only structure** into `.cpp` files — the pattern is
  intentional, consistent, and avoids build-system complexity for small samples.
- **Do not add a shared `samples/common/` library** — each sample must be self-contained.
- **Do not change `cna_add_sample()`** without auditing all 12 existing `CMakeLists.txt`.
- **Do not re-generate existing font atlases** (SafeArea, InputSequence, InputReporter,
  Platformer) unless there is a confirmed rendering bug — regenerating changes pixel
  layout and may break visual output.
- **Do not mass-update PLAN.md statuses** in a single commit — update one sample's
  status per commit to keep history readable.
