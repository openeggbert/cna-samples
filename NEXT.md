# NEXT.md — CNA Samples Handoff

## 1. Project Summary

**What:** C++ ports of the official Microsoft XNA Game Studio 4.0 sample collection,
running on CNA — a C++ reimplementation of the XNA 4.0 API built on SDL3.

**Main goal:** Port all ~83 applicable XNA 4.0 desktop samples to CNA C++, preserving
the original class hierarchy and naming (`Microsoft::Xna::Framework::*`), serving as
living integration tests for the CNA framework and a migration reference.

**Current phase:** Phase 1 (Foundation) and Phase 2 (2D Games) are fully complete
(excluding items deferred due to missing CNA features). Phase 5 (Audio) has one of
two samples done. Phases 3, 4, 6, 7 are untouched.

**Key architectural decisions:**
- One executable per sample; no shared sample library.
- All sample code is header-only (except `Program.cpp`).
- Assets: `Load<Texture2D>()` → PNG; `Load<SpriteFont>()` → `.font.json` + PNG atlas
  (generated by `tools/make_font.py`); `Load<SoundEffect>()` → WAV;
  `Load<Song>()` → OGG/WAV/MP3/FLAC; `Load<Model>()` → `.model.json` format.
- `CONTENT_DIR` argument to `cna_add_sample()` is mandatory for any sample that
  loads assets at runtime.
- Two configured build trees: `cmake-build-debug` (Vulkan) and `cmake-build-easygl` (EasyGL).
  Both trees build all enabled samples successfully.

### What does not work
- **CameraShake 3D scene** renders as a white stripe on all CNA backends (Vulkan and EasyGL).
  The XNA original camera at (1000,1000,1000) with ground scale 0.1 places the corner
  vertex at (6554,0,6554) behind the camera — CNA near-plane clipping of w<0 vertices
  produces an artefact instead of the clean DirectX clip. See `samples/CameraShake/missing.md`.
- `BasicEffect.DiffuseColor` silently ignored for `VertexPositionColor` geometry (EasyGL).
- `FillMode::WireFrame` silently ignored on OpenGL ES (EasyGL).
- All Phase 3+ samples with custom HLSL shaders require manual GLSL translation before porting.

## 4. Current Blocker / Main Problem

**CameraShake white stripe (all backends).**

- **Symptom:** The 3D scene in CameraShake renders as a white stripe only. Tank and ground
  are invisible; only a thin bright bar appears.
- **Affected files:** `samples/CameraShake/src/CameraShakeGame.hpp` (camera setup at line 112),
  `samples/CameraShake/Content/ground_verts.bin` (ground geometry).
- **Root cause:** Camera at (1000,1000,1000) with ground scale 0.1 → ground vertices at ±6554.
  Vertex at (6554,0,6554) has x+z=13108 which is behind the camera (threshold ≈3000 for this
  camera position). CNA near-plane clipping of w<0 vertices does not match DirectX behaviour.
- **Tried:** Removed wrong z-remap from Vulkan shaders (correct fix, no visual change).
  Tested scale 0.02 workaround (avoids clip, but was reverted to keep XNA original values).
  Tested EasyGL backend — same white stripe.
- **Fix path:** Either fix CNA's near-plane clipping, or apply a sample-level workaround
  (scale, camera distance, or ground mesh subdivision).

## 6. Architecture Notes

### Sample directory layout
```
samples/SampleName/
  src/
    Program.cpp          # int main() { Game g; g.Run(); return 0; }
    GameClass.hpp        # all logic inline
  Content/               # copied next to binary via CONTENT_DIR
  CMakeLists.txt         # cna_add_sample(name SOURCES src/Program.cpp CONTENT_DIR ...)
  missing.md             # required: differences vs XNA original
  SampleName.htm         # required: original MS documentation, verbatim copy
```

### CNA API rules — must not break
- All XNA properties: `getXxxProperty()` / `setXxxProperty()` — never raw member access.
- Every `Game` subclass must implement:
  ```cpp
  const std::string& GetTypeName() const override {
      static const std::string name = "ClassName";
      return name;
  }
  ```
- `Content.Load<T>()` returns **T by value** — store in `std::optional<T>` field.
- `SpriteBatch::Draw` takes `const Texture2D&` — use `*optionalTex`.
- Virtual override signatures: `void Update(GameTime& gameTime)` and
  `void Draw(const GameTime& gameTime)` — note const on Draw.
- No `operator+=` for `Vector2` or `TimeSpan` — write `v = v + expr`.
- `SoundEffect` has no default constructor — use `.emplace()` not `operator[]` in maps.
- `SoundEffectInstance` is move-only — wrap in `std::unique_ptr<SoundEffectInstance>`.
- `SoundEffect::CreateInstance()` returns by value → move into unique_ptr.
- `AudioListener`/`AudioEmitter` use property setters: `listener.setPositionProperty(pos)`.
- `GameComponent` subclasses: call base `Initialize()` / `Update()` at end of overrides.
- `getComponentsProperty().Add(ptr)` to register components with Game.

### SpriteFont generation
```bash
python3 tools/make_font.py /path/to/Font.ttf <size_px> \
    samples/X/Content/Fonts/FontName \
    --content-root samples/X/Content
```
**Always pass `--content-root samples/X/Content`.** Without it the `.font.json`
records a bare filename (`"FontName.png"`) instead of the path relative to the
Content root (`"Fonts/FontName.png"`), and ContentManager fails at runtime.

### Sound assets
- `SoundEffect` loads `.wav` only.
- `Song` (music) loads: `.mp3 .ogg .wav .flac .opus .aac .wma`

### Help overlay (F1)
```bash
python3 tools/gen_help_png.py samples/SampleName/SampleName.htm samples/SampleName/Content/help.png
```

## 9. Do Not Do

- **Do not start Phase 3 samples without translating their HLSL shaders first** — the custom
  Effect loader expects `.shader.json` + GLSL; no HLSL runtime compiler exists in CNA.
- **Do not add a shared `samples/common/` library** — each sample must be self-contained.
- **Do not re-generate existing font atlases** unless there is a confirmed rendering bug.
- **Do not change the ground scale in CameraShake** without first fixing the CNA near-plane
  clipping bug — a workaround in the sample hides a real CNA issue.
- **Do not mass-commit PLAN.md** — update only the rows that actually changed.
- **Do not investigate Vulkan-specific fixes for CameraShake** — the same bug exists on EasyGL;
  fix belongs in shared geometry/clipping logic.
