# Missing / Differences from XNA 4.0 original

## Font: DejaVu Sans instead of Arial

**XNA behaviour:** Uses Arial (Regular/Bold) at 10–28pt.

**CNA port behaviour:** Uses DejaVu Sans (Regular/Bold) at equivalent pixel sizes
generated by `tools/make_font.py`. Layout positions match the original exactly.

**Root cause:** Arial is not freely distributable; DejaVu Sans is the closest
open-source substitute.

**Tracked in:** not planned (acceptable substitute).

---

## Path separator: `/` instead of `\`

**XNA behaviour:** Font assets loaded as `"Fonts\\DataFont"` (Windows path separator).

**CNA port behaviour:** Loaded as `"Fonts/DataFont"` (POSIX path separator).

**Root cause:** CNA ContentManager on Linux uses `/`.

**Tracked in:** not planned (platform convention).

---

## PlayerIndex displayed as XNA enum name

**XNA behaviour:** `((PlayerIndex)selectedPlayer).ToString()` produces `"One"`, `"Two"`, etc.

**CNA port behaviour:** Same — `PlayerIndexToString()` returns `"One"`, `"Two"`, `"Three"`, `"Four"`.

**Root cause:** C++ enums don't stringify automatically; helper added.

**Tracked in:** not planned (output identical).

---

## No known differences otherwise

ChargeSwitch hold-to-fire logic, dead-zone toggle (Space / hold START), exit
(Escape / hold BACK), per-player input display, all data columns, and the
connected/selected controller images all match the XNA 4.0 original.
